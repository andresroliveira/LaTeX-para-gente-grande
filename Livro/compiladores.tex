\chapter{Compiladores, Fontes e Automação}

A compreensão profunda do processo de compilação é o que diferencia o usuário casual do \LaTeX\ de um editor técnico profissional. Frequentemente, frustrações comuns --- como referências cruzadas que aparecem como ``??'', sumários vazios ou erros de fontes não reconhecidas --- derivam de uma visão superficial do funcionamento interno do sistema. Entender que o editor não é o compilador e que o \LaTeX\ é um ecossistema modular é o primeiro passo para dominar a ferramenta.

\section{Ecossistema: Distribuição, Editor, Compilador, Classe e Pacote}

O \LaTeX\ não é um programa único, mas um conjunto de componentes interdependentes que interagem para gerar o documento final.

\begin{itemize}
    \item \textbf{Distribuição TeX}: É o repositório completo de ferramentas. As principais são o \textbf{TeX Live} (padrão em Linux e macOS) e o \textbf{MiKTeX} (Windows). Em ambientes online, o \textbf{Overleaf} fornece uma distribuição gerenciada na nuvem.
    \item \textbf{Editor}: A interface de escrita. Exemplos incluem o \textbf{TeXstudio} (IDE dedicada), o \textbf{VS Code} (com LaTeX Workshop), o \textbf{Vim} (com VimTeX) e o \textbf{Emacs} (com \textbf{AUCTeX}). O editor apenas facilita a escrita e invoca o compilador.
    \item \textbf{Compilador}: O programa binário que processa o código. No ecossistema TeX, esses programas são frequentemente chamados de \textbf{motores} ou \textit{engines}. Exemplos: \texttt{pdflatex}, \texttt{xelatex}, \texttt{lualatex}.
    \item \textbf{Classe (\texttt{.cls})}: Define o esqueleto e o layout base (ex: \texttt{book}, \texttt{article}, \texttt{beamer}).
    \item \textbf{Pacote (\texttt{.sty})}: Extensões de funcionalidade (ex: \texttt{amsmath}, \texttt{graphicx}, \texttt{tikz}).
\end{itemize}

\section{Fluxo de Compilação e Arquivos Auxiliares}

Quando executamos o comando \texttt{pdflatex arquivo.tex}, o compilador realiza uma leitura sequencial. No entanto, elementos que dependem da estrutura global (como sumários e referências) não podem ser resolvidos em uma única passagem.

\subsection{Por que múltiplas compilações?}
Considere uma referência cruzada para um capítulo que está dez páginas à frente. Na primeira passagem, o compilador ainda não sabe em qual página aquele capítulo terminará. Ele então anota o nome do \textit{label} em um arquivo auxiliar e continua. Na segunda passagem, ele lê esse arquivo auxiliar e substitui a marcação pelo número correto.

\subsection{Anatomia dos Arquivos Gerados}
\begin{itemize}
    \item \textbf{\texttt{.aux}}: Armazena labels, citações e contadores. É a ``memória'' entre passagens.
    \item \textbf{\texttt{.log}}: Registro detalhado de tudo o que ocorreu. É o primeiro lugar a ser consultado em caso de erro.
    \item \textbf{\texttt{.toc, .lof, .lot}}: Estruturas para sumário, figuras e tabelas.
    \item \textbf{\texttt{.bbl}}: Gerado pelo BibTeX/Biber para armazenar a bibliografia formatada.
\end{itemize}

\section{Compiladores: pdflatex, xelatex e lualatex}

A escolha do compilador determina como o sistema lida com fontes e caracteres modernos.

\begin{enumerate}
    \item \textbf{\texttt{pdflatex}}: O padrão estável. Usa o modelo tradicional de fontes do TeX (\textit{Type 1}). É rápido e amplamente compatível, mas exige pacotes específicos para carregar diferentes fontes.
    \item \textbf{\texttt{xelatex}}: Suporta nativamente Unicode (UTF-8) e permite o uso de fontes instaladas no sistema operacional (OTF/TTF) via pacote \texttt{fontspec}.
    \item \textbf{\texttt{lualatex}}: Motor moderno baseado em LuaTeX, com suporte a Unicode, gestão de fontes via \texttt{fontspec} e possibilidade de extensão de funcionalidades através da linguagem Lua.
\end{enumerate}

\section{Fontes no pdfLaTeX}

Este projeto utiliza o \texttt{pdflatex} pela sua robustez em documentos técnicos. Para obter resultados profissionais em português, configuramos o tripé de codificação:

\begin{itemize}
    \item \textbf{Entrada (UTF-8)}: Garante que o arquivo \texttt{.tex} salve acentos corretamente. Ativado por \verb|\usepackage[utf8]{inputenc}| (padrão em versões recentes).
    \item \textbf{Saída (T1)}: A codificação de fonte \texttt{T1} (habilitada por \verb|\usepackage[T1]{fontenc}|) é obrigatória para o português. Ela garante que o PDF contenha glifos acentuados únicos, permitindo a hifenação correta e a busca de texto (ex: buscar ``ação'' e encontrar o resultado, em vez de ``ac'' e ``~ao'' separados).
    \item \textbf{Pacotes de Fontes}: Como o \texttt{pdflatex} não carrega arquivos \texttt{.ttf} diretamente, usamos pacotes como \texttt{ebgaramond} ou \texttt{newpxtext} que encapsulam as métricas necessárias.
\end{itemize}

\section{Microtipografia com microtype}

O pacote \texttt{microtype} introduz ajustes finos que reduzem a fadiga visual:
\begin{itemize}
    \item \textbf{Protrusão}: Ajusta pontuações nas margens para criar uma borda visual reta.
    \item \textbf{Expansão}: Ajusta a largura dos caracteres para evitar hifenações excessivas e buracos brancos no texto.
\end{itemize}

\section{O que é fontspec (Comparativo)}

\textit{Nota: O exemplo abaixo não é compatível com o pdflatex e serve apenas para demonstração didática do uso de fontes do sistema em XeLaTeX ou LuaLaTeX.}

\begin{lstlisting}[language={[latex]TeX}, caption={Uso de fontspec}]
\usepackage{fontspec}
\setmainfont[
    Path = fontes/,
    UprightFont = MinhaFonte-Regular.otf,
    BoldFont = MinhaFonte-Bold.otf
]{MinhaFonte}
\end{lstlisting}
Neste código, o \texttt{fontspec} permite selecionar arquivos locais (\texttt{Path}) ou do sistema pelo nome real, facilitando a portabilidade em outros motores.

\section{Automação e Fluxos de Trabalho}

A execução manual de comandos como \texttt{pdflatex}, \texttt{bibtex}, \texttt{pdflatex}, \texttt{pdflatex} é propensa a erros.

\begin{itemize}
    \item \textbf{\texttt{latexmk}}: É a ferramenta recomendada. Ela monitora mudanças nos arquivos e executa o ciclo de compilação automaticamente.
    \item \textbf{Makefile}: Neste projeto, fornecemos um \texttt{Makefile} para que o comando \texttt{make} sirva como uma interface conveniente, abstraindo as chamadas complexas do terminal.
    \item \textbf{Fluxos Integrados}: Usuários de Overleaf ou IDEs como TeXstudio já possuem esse ciclo configurado por botões visuais, mas entender o que ocorre ``embaixo dos panos'' é vital para resolver falhas de build.
\end{itemize}

\section{Erros e Diagnóstico: Lendo o Log}

A primeira regra da depuração em \LaTeX\ é: \textbf{ignore os últimos erros e foque no primeiro}. Frequentemente, um único erro de sintaxe gera uma cascata de falhas subsequentes.

\begin{description}
    \item[\texttt{Undefined control sequence}] Comando inexistente ou pacote não carregado. Verifique se digitou \verb|\section| em vez de \verb|\secton|.
    \item[\texttt{File not found}] O compilador não encontrou uma imagem ou arquivo incluído. Verifique caminhos e extensões.
    \item[\texttt{Missing \$ inserted}] Ocorreu o uso de um símbolo matemático (como \_) fora do modo matemático.
    \item[\texttt{Overfull \textbackslash hbox}] O texto excedeu a margem lateral. Ajuste a hifenação ou reformule o parágrafo.
\end{description}

\section{Consulta de Documentação}

A documentação local é a fonte mais confiável. O comando \texttt{texdoc} abre o manual oficial de qualquer pacote instalado:
\begin{lstlisting}[language=bash]
texdoc amsmath
texdoc graphicx
texdoc beamer
\end{lstlisting}
Para buscas globais e versões mais recentes, utilize o \textbf{CTAN} (\url{https://ctan.org}).
